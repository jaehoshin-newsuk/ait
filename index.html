                    <h3 class="text-xl font-bold text-[#073B4C] mb-2">CTE 1: `unified_spec`</h3>

                    <p class="text-gray-600">Loads the "source of truth" from a BigQuery table populated by a Google Sheet. This defines every field we expect to see for each interaction type.</p>

                </div>

                <div class="bg-white p-6 rounded-lg shadow-lg border-t-4 border-[#06D6A0]">

                    <h3 class="text-xl font-bold text-[#073B4C] mb-2">CTE 2: `raw_events_...`</h3>

                    <p class="text-gray-600">Ingests and classifies raw clickstream data using `CASE WHEN` logic, assigning a business-friendly `interaction_type` to each event.</p>

                </div>

                <div class="bg-white p-6 rounded-lg shadow-lg border-t-4 border-[#FFD166]">

                    <h3 class="text-xl font-bold text-[#073B4C] mb-2">CTE 3: `raw_data_unpivoted`</h3>

                    <p class="text-gray-600">Performs a dynamic "unpivot" by joining the spec to the raw data and using a large `CASE` statement to create a normalized `(field_name, field_value)` structure.</p>

                </div>

                <div class="bg-white p-6 rounded-lg shadow-lg border-t-4 border-[#FF6B6B]">

                    <h3 class="text-xl font-bold text-[#073B4C] mb-2">Final SELECT</h3>

                    <p class="text-gray-600">Joins the spec against aggregated data, using `COALESCE` to clean up NULLs and a final `CASE` statement to assign an `audit_status` to each field.</p>

                </div>

            </div>
